对全部 18 个 Pydantic schema 文件执行 PEP 604 降级（per D-01, D-03）：

**在 BaseModel 字段定义（类体内的变量赋值行）中替换：**
- `str | None` → `Optional[str]`
- `int | None` → `Optional[int]`
- `float | None` → `Optional[float]`
- `bool | None` → `Optional[bool]`
- `datetime | None` → `Optional[datetime]`
- `date | None` → `Optional[date]`
- 任何 `TypeName | None` 形式同理（包括自定义类型如 `SomeEnum | None` → `Optional[SomeEnum]`）

**关键区分规则（per D-02）：**
- Pydantic BaseModel 的字段定义（类体内，如 `name: str | None = None`）是运行时求值 → 必须降级
- 函数参数和返回值注解（如 `def foo(x: str | None) -> str | None:`）是纯注解 → 不降级
- `from __future__ import annotations` 保护函数签名中的注解，但 Pydantic 字段通过 `model_fields` 在运行时评估类体内的注解

**判断方法：** 如果一行在 `class XxxModel(BaseModel):` 或 `class XxxSchema(BaseModel):` 等 Pydantic 类体内部，且形如 `field_name: SomeType | None = ...`，则必须降级。如果在函数 `def` 内部或作为函数参数，则不降级。

**导入添加：**
- 在每个被修改文件中添加 `from typing import Optional`
- 如果文件已有 `from typing import ...`，在现有导入中追加 `Optional`
- 保持导入排序：`__future__` → stdlib/typing → third-party → local
  </action>
