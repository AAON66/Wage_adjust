**PEP 585 降级（运行时求值位置）：**
- `Mapped[dict[str, ...]]` → `Mapped[Dict[str, ...]]`（仅 employee_handbook.py 有 2 处）

- 在每个被修改文件的 `from __future__ import annotations` 之后、其他导入之前，添加 `from typing import Optional`

在 `backend/app/core/database.py` 中添加 SQLite 外键约束启用（per D-07, D-08）：

1. 在文件顶部的 sqlalchemy 导入中添加 `from sqlalchemy import event`（与现有 `from sqlalchemy import MetaData, create_engine` 合并）

2. 在 `create_db_engine()` 函数内、`return create_engine(...)` 之前添加 SQLite FK event listener：

```python
def _set_sqlite_pragma(dbapi_conn, connection_record):
    """Enable foreign key enforcement for SQLite connections."""
    cursor = dbapi_conn.cursor()
    cursor.execute('PRAGMA foreign_keys=ON')
    cursor.close()
```

3. 修改 `create_db_engine()` 为：

```python
def create_db_engine(settings: Settings | None = None) -> Engine:
    """Build a SQLAlchemy engine from application settings."""
    resolved_settings = settings or get_settings()
    new_engine = create_engine(resolved_settings.database_url, **_engine_kwargs(resolved_settings))
    if resolved_settings.database_url.startswith('sqlite'):
        event.listen(new_engine, 'connect', _set_sqlite_pragma)
    return new_engine
```

注意：`_set_sqlite_pragma` 定义在模块级别（`create_db_engine` 之前），不是函数内部。仅对 SQLite 连接启用（per D-08），PostgreSQL 不受影响。

同时注意 `create_db_engine` 的参数注解 `Settings | None = None` 是纯注解位置（函数签名），被 `from __future__ import annotations` 保护，不需要降级。
