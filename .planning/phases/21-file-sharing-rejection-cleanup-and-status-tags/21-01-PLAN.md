## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `POST /sharing-requests/{id}/reject` -> `SharingService.reject_request()` | 授权通过后会触发 requester 副本删除，目标选择和事务边界都不能受客户端输入污染 |
| `SharingService` -> `FileService` delete helper | 一个状态流转跨越 DB 行、evidence 和物理存储，若 helper 内部自提交会出现半完成 cleanup |
| `uploaded_files` FK delete -> `sharing_requests` history row | requester 副本删除若仍触发 cascade，会直接抹掉审计历史 |
| `GET /submissions/{submission_id}/files` -> lazy expiry cleanup | 文件列表读取会顺带触发 timeout cleanup，必须只影响当前合法目标，不得泄漏或误删 |

| T-21-02 | Repudiation / Tampering | `backend/app/models/sharing_request.py` requester FK + history query coupling | mitigate | 把 `requester_file_id` 改为 history-safe 持久策略，持久化 `requester_content_hash`（及必要快照字段），并将 history / duplicate-precheck 改走持久键 |
