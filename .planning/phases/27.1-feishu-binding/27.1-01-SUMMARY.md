# Phase 27.1 Plan 01: FeishuOAuthService bind/unbind 服务层扩展 Summary

扩展 `FeishuOAuthService` 以支持登录后的「飞书账号绑定 / 解绑」能力，通过 state 键前缀物理隔离 login / bind 流程，并在绑定路径强制校验工号一致性与 open_id 唯一占用。

---

## What Was Built

### Task 1: `feishu_bind_redirect_uri` Settings 字段
- `backend/app/core/config.py`：新增 `feishu_bind_redirect_uri: str = ''` 字段，紧挨既有 `feishu_redirect_uri` 以保持配置分组视觉对齐
- `.env.example`：新增 `FEISHU_BIND_REDIRECT_URI=http://localhost:5174/settings/feishu-bind-callback` 示例行
- 空字符串作为哨兵值：Plan 02 的 authorize endpoint 在 `purpose=bind` 且值为空时返回 503

### Task 2: `FeishuOAuthService` bind / unbind 能力
- **`generate_authorize_url(redis_client, *, purpose='login')`**：扩展签名接受关键字专有参数 `purpose`
  - `purpose='login'`（默认）：state key `feishu_oauth_state:{state}`，redirect_uri 用 `feishu_redirect_uri` — **登录路径完全向后兼容**
  - `purpose='bind'`：state key `feishu_oauth_state:bind:{state}`，redirect_uri 用 `feishu_bind_redirect_uri`；空值 → 503
  - 未知 purpose → 400 `'无效的 purpose 参数'`
- **`_validate_state_bind(state, redis_client)`**：与 `_validate_state` 平行的 bind 版，仅读 `feishu_oauth_state:bind:` 前缀的键 — login state 物理上无法被 bind 路径消费，反之亦然
- **`handle_bind_callback(code, state, user, redis_client) -> User`**（新增公开方法）：
  1. `_validate_state_bind` 消费 bind 前缀 state
  2. `_check_code_replay` 防重放（与 login 共享 code 防重放空间）
  3. `_exchange_code_for_token_for_bind`（绑定专用，见下）换 access_token
  4. `_get_user_info` 获取 open_id + employee_no
  5. **D-07**：`user.employee_id is None` 或 `user.employee is None` → 400 `'当前账号未关联员工信息，请联系管理员'`
  6. **D-06**：飞书 employee_no 与 `user.employee.employee_no` 不一致（leading-zero 容差后）→ 400 `'飞书工号与当前账号绑定的工号不一致，无法绑定'`
  7. **D-05**：`select(User).where(User.feishu_open_id == open_id, User.id != user.id)` 命中 → 409 `'该飞书账号已被其他系统账号绑定，请先在原账号解绑'`
  8. 全部通过 → `user.feishu_open_id = open_id; db.flush()`；返回 user
- **`_exchange_code_for_token_for_bind(code)`**（新增私有方法）：与 `_exchange_code_for_token` 的唯一差异是 `redirect_uri` 取 `feishu_bind_redirect_uri`。**故意不合并**，以保持 Phase 26 登录路径字节级稳定、零回归风险
- **`handle_unbind(user) -> User`**（新增公开方法）：
  - 若 `user.feishu_open_id is not None` → 置 None + `db.flush()`
  - 否则幂等无动作
  - **不触碰 `token_version`**（D-10：保留当前会话，解绑不踢用户下线）

## Plan 02 依赖接口（verbatim 契约）

Plan 02（API 路由层）MUST 调用下列签名：

```python
# backend/app/services/feishu_oauth_service.py

class FeishuOAuthService:
    def generate_authorize_url(
        self,
        redis_client: redis_lib.Redis,
        *,
        purpose: str = 'login',  # 'login' | 'bind'
    ) -> dict[str, str]: ...  # {'authorize_url': str, 'state': str}

    def handle_bind_callback(
        self,
        code: str,
        state: str,
        user: User,
        redis_client: redis_lib.Redis,
    ) -> User: ...  # HTTPException 400 / 409 / 502 / 503

    def handle_unbind(self, user: User) -> User: ...  # idempotent
```

Plan 02 预期扩展点：
- `GET /api/v1/auth/feishu/authorize` 接受 `?purpose=login|bind` 查询参数，转发给 service
- `POST /api/v1/auth/feishu/bind`（需 JWT）调 `handle_bind_callback`
- `POST /api/v1/auth/feishu/unbind`（需 JWT）调 `handle_unbind`
- AuditLog 写入（`feishu_bound` / `feishu_unbound`）在 handler 层完成；open_id 截断为前 8 位 + 后 8 位
