    **1. 创建 `backend/app/core/redis.py`（Redis 连接管理）：**
    - 使用 `redis.from_url(settings.redis_url, decode_responses=True)` 创建全局连接池
    - 提供 `get_redis() -> redis.Redis` 函数，懒加载单例模式

    **2. 创建 `backend/app/services/cache_service.py`（Redis 缓存封装）：**
    - `CacheService.__init__(self, redis_client: redis.Redis | None)`
    - `_build_key(chart_type, cycle_id, role) -> str`: key 格式 `dashboard:{cycle_id or 'all'}:{role}:{chart_type}`（per D-04）
    - `get(chart_type, cycle_id, role) -> dict | list | None`: 缓存命中返回 JSON，未命中返回 None。redis_client 为 None 时直接返回 None。
    - `set(chart_type, cycle_id, role, data, ttl_seconds=300)`: 序列化写入。使用 `json.dumps(data, default=str)` 处理 Decimal。redis_client 为 None 时静默跳过。
    - `invalidate_cycle(cycle_id)`: 删除该周期所有缓存 key（使用 `keys()` 模式匹配，项目规模小可接受）。
    - TTL 常量定义：
      - `TTL_KPI_SUMMARY = 0`（不缓存，30 秒前端轮询直接查询）
      - `TTL_AI_LEVEL = 600`（10 分钟）
      - `TTL_SALARY_DIST = 600`（10 分钟）
      - `TTL_APPROVAL_PIPELINE = 300`（5 分钟）
      - `TTL_DEPARTMENT_INSIGHT = 900`（15 分钟）
      - `TTL_DEPARTMENT_DRILLDOWN = 600`（10 分钟）

    **3. 改造 `backend/app/services/dashboard_service.py`：**

    保留现有类结构和 `AI_LEVEL_ORDER`、`ACTIVE_RECOMMENDATION_STATUSES`、`REVIEW_BACKLOG_STATUSES` 常量。

    新增方法（不删除旧方法，新旧并存便于渐进切换）：

    a. `_department_filter(self, current_user: User | None) -> set[str] | None`:
       - `current_user is None` 或 `current_user.role in {'admin', 'hrbp'}` 返回 `None`（无过滤）
       - `current_user.role == 'manager'` 返回 `AccessScopeService(self.db)._department_names(current_user)`
       - 其他角色返回空 set（看不到数据，per D-11 员工不可见看板）

    b. `get_ai_level_distribution_sql(self, cycle_id: str | None, department_filter: set[str] | None) -> list[dict]`:
       ```python
       from sqlalchemy import func, select
       query = (
           select(AIEvaluation.ai_level, func.count(AIEvaluation.id).label('count'))
           .join(AIEvaluation.submission)
           .join(EmployeeSubmission.employee)
       )
       if cycle_id:
           query = query.where(EmployeeSubmission.cycle_id == cycle_id)
       if department_filter is not None:
           query = query.where(Employee.department.in_(department_filter))
       query = query.group_by(AIEvaluation.ai_level)
       rows = self.db.execute(query).all()
       count_map = {row.ai_level: row.count for row in rows}
       total = sum(count_map.values())
       return [
           {'label': level, 'value': count_map.get(level, 0),
            'percentage': round(count_map.get(level, 0) / total * 100, 1) if total > 0 else 0}
           for level in self.AI_LEVEL_ORDER
       ]
       ```

    c. `get_salary_distribution_sql(self, cycle_id: str | None, department_filter: set[str] | None) -> list[dict]`:
       - 使用 `case()` 将 `final_adjustment_ratio` 分桶为 '0-5%', '5-10%', '10-15%', '15-20%', '20%+'
       - JOIN 链: `SalaryRecommendation -> AIEvaluation -> EmployeeSubmission -> Employee`
       - WHERE: `SalaryRecommendation.status.in_(self.ACTIVE_RECOMMENDATION_STATUSES)`
       - 可选 cycle_id 和 department_filter
       - GROUP BY bucket

    d. `get_approval_pipeline_sql(self, cycle_id: str | None, department_filter: set[str] | None) -> list[dict]`:
       - GROUP BY `SalaryRecommendation.status`
       - JOIN 链同上
       - 返回 `[{'label': status, 'value': count}, ...]`

    e. `get_kpi_summary_sql(self, cycle_id: str | None, department_filter: set[str] | None) -> dict`:
       - 待审批数: `SELECT COUNT(*) FROM salary_recommendations WHERE status = 'pending_approval'`（加 cycle_id 和 department 过滤）
       - 员工统计: 总人数 `SELECT COUNT(DISTINCT employee_id) FROM employee_submissions`，已评估数 `SELECT COUNT(*) FROM ai_evaluations`（加过滤）
       - 平均调薪幅度: `SELECT AVG(final_adjustment_ratio) FROM salary_recommendations WHERE status IN (...)`
       - 等级概览: 复用 `get_ai_level_distribution_sql` 结果取前 3 名
       - 返回: `{'pending_approvals': int, 'total_employees': int, 'evaluated_employees': int, 'avg_adjustment_ratio': float, 'level_summary': list}`

    f. `get_department_drilldown_sql(self, department: str, cycle_id: str | None) -> dict`:
       - 该部门的 AI 等级分布: GROUP BY ai_level WHERE department = ?
       - 该部门的平均调薪幅度: AVG(final_adjustment_ratio)
       - 返回: `{'department': str, 'level_distribution': list, 'avg_adjustment_ratio': float, 'employee_count': int}`
