2. Replace `serialize_recommendation()` with `shape_recommendation_for_role()`:
```python
def shape_recommendation_for_role(
    recommendation,
    role: str,
) -> SalaryRecommendationAdminRead | SalaryRecommendationEmployeeRead:
    """Return role-appropriate salary recommendation response per D-13.

    admin, hrbp → full figures (SalaryRecommendationAdminRead)
    manager, employee → adjustment percentage only (SalaryRecommendationEmployeeRead)
    """
    if role in ('admin', 'hrbp'):
        return SalaryRecommendationAdminRead(
            id=recommendation.id,
            evaluation_id=recommendation.evaluation_id,
            current_salary=recommendation.current_salary,
            recommended_ratio=recommendation.recommended_ratio,
            recommended_salary=recommendation.recommended_salary,
            ai_multiplier=recommendation.ai_multiplier,
            certification_bonus=recommendation.certification_bonus,
            final_adjustment_ratio=recommendation.final_adjustment_ratio,
            status=recommendation.status,
            created_at=recommendation.created_at,
            explanation=getattr(recommendation, 'explanation', None),
        )
    return SalaryRecommendationEmployeeRead(
        id=recommendation.id,
        evaluation_id=recommendation.evaluation_id,
        final_adjustment_ratio=recommendation.final_adjustment_ratio,
        status=recommendation.status,
        created_at=recommendation.created_at,
        explanation=getattr(recommendation, 'explanation', None),
    )
```

4. Update the `response_model` on all 3 routes. Since the response type is a union, use `response_model=SalaryRecommendationRead` (keep existing for backwards compat) BUT have FastAPI return the actual shaped model. Alternatively, use `Union[SalaryRecommendationAdminRead, SalaryRecommendationEmployeeRead]` as the response model.

CLEANEST APPROACH: Keep `response_model=SalaryRecommendationRead` for the OpenAPI schema (it contains the full set of fields), but return the shaped model from the function. FastAPI will serialize only the fields present in the returned Pydantic model. Since `SalaryRecommendationEmployeeRead` lacks `current_salary` etc., FastAPI will NOT include them in the response (Pydantic serialization only includes fields defined on the model being serialized).

This means no `response_model` change is needed — FastAPI serializes the returned object using its own schema, not the declared `response_model`, when `response_model` is a supertype.

ACTUALLY: FastAPI serializes by filtering the return value through `response_model`. If `response_model=SalaryRecommendationRead` (which includes `current_salary`) and you return `SalaryRecommendationEmployeeRead` (which lacks `current_salary`), FastAPI will call `model.model_dump()` on the return value and validate it against `response_model`. This WILL fail for the employee schema because `current_salary` is required in `SalaryRecommendationRead`.

CORRECT APPROACH: Change `response_model` on all 3 affected routes to `response_model=None` (disables validation) and return the Pydantic model directly. FastAPI will serialize it using the model's own schema.

OR: Use `Union`:
```python
from typing import Union
# response_model=Union[SalaryRecommendationAdminRead, SalaryRecommendationEmployeeRead]
```

USE `response_model=None` for simplicity and to avoid OpenAPI union complexity. FastAPI returns a JSONResponse by serializing the returned Pydantic model.

Final route signature for the 3 affected routes:
```python
@router.post('/recommend', status_code=status.HTTP_201_CREATED)
def recommend_salary(...) -> SalaryRecommendationAdminRead | SalaryRecommendationEmployeeRead:
    ...
    return shape_recommendation_for_role(recommendation, current_user.role)
```
