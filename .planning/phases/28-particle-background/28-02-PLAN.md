```typescript
export function ParticleBackground(): JSX.Element | null;
// Props: 无
// 自管理: 生命周期、事件、rAF、reduced-motion、visibility、HiDPI
// DOM: 输出一个 <canvas> 带 position:fixed/inset:0/z-index:0/pointer-events:none/aria-hidden
// Fallback: prefers-reduced-motion: reduce 时 return null
```

Login.tsx 现状（简化）:
- 顶层 `<main>` 使用内联样式 `background: 'var(--color-bg-page)'`
- 内部两个 `<section className="surface animate-fade-up">` 作为卡片
- LoginForm / FeishuLoginPanel / Link 均为交互元素
- 无任何 z-index 显式设置，浏览器默认 z-index: auto

关键约束:
- Canvas 以 z-index: 0 + pointer-events: none 挂载（Plan 01 已实现）
- `<main>` 和 `<section>` 必须 z-index >= 1，否则虽然 pointer-events:none 不拦截点击，但视觉层级上若卡片出现半透明/blend 场景可能被 Canvas 色调影响。本 Plan 给 `<main>` 显式 `position: relative; zIndex: 1` 作为稳健选择（per D-03 后半段：登录主内容 z-index 提升到 ≥ 1）
