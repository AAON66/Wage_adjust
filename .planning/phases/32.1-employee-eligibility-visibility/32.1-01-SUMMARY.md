## Endpoint Signature

```python
@router.get('/me', response_model=EligibilityResultSchema)
def get_my_eligibility(
    reference_date: date | None = None,
    year: int | None = None,
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user),
) -> EligibilityResultSchema
```

**Error codes:**
- `200` 已绑定 employee 的 user → EligibilityResultSchema(overall_status, rules[4], data_updated_at)
- `401` 未鉴权（来自 oauth2_scheme）
- `422` user.employee_id is None → `{error: 'http_error', message: '您尚未绑定员工信息，请前往「账号设置」完成绑定'}`
- `404` user.employee_id 指向已删除/不存在 employee → `{error: 'http_error', message: '员工未找到'}`

## Decisions Made

- **未绑定 user 返 422 而非 401/200**：明确告诉前端「鉴权 OK 但业务态不满足」，附中文 detail 引导用户去「账号设置」完成绑定
- **悬挂 employee_id 复用 check_employee 的 404**：不在 /me 内重复 employee 查找，保持单一职责
- **测试 T3 用 raw SQL UPDATE 模拟悬挂**：ORM `db.delete(emp)` 会触发关系刷新把 `user.employee_id` 置 None，绕开 ORM 才能模拟「user 没解绑但员工被删」的生产场景
