## LoadState 状态机代码段

```typescript
type LoadState =
  | { kind: 'loading' }
  | { kind: 'success'; data: EligibilityResultWithTimestamp }
  | { kind: 'unbound' }              // HTTP 422
  | { kind: 'employee_missing' }      // HTTP 404
  | { kind: 'error'; message: string }; // 500/network/其他

const load = useCallback(async () => {
  setState({ kind: 'loading' });
  try {
    const data = await fetchMyEligibility();
    setState({ kind: 'success', data });
  } catch (err) {
    if (axios.isAxiosError(err)) {
      const status = err.response?.status;
      if (status === 422) { setState({ kind: 'unbound' }); return; }
      if (status === 404) { setState({ kind: 'employee_missing' }); return; }
    }
    setState({ kind: 'error', message: '资格信息暂时不可用，请稍后重试' });
  }
}, []);
```

## Decisions Made

- **discriminated union 状态机**（loading/success/unbound/employee_missing/error）穷举所有可能态，TS 编译器强制 JSX 渲染分支覆盖；避免多个 boolean 标志混乱
- **panel 独立于 employee 三元式**：`<MyEligibilityPanel />` 顶层渲染于 line 545，不嵌入 `{employee ? ... : null}`；panel 通过 user.employee_id 决定能否拿到资格，与 MyReview 现有 `findEmployeeForUser` 解耦
- **错误态分流不抛 toast**：422/404/500 均渲染卡片在 panel 内，不打扰 MyReview 其他 section（保持页面其他功能可用）
- **data-testid 命名带状态后缀**：`overall-badge-{status}` / `rule-row-{code}` 便于 Playwright 用单选择器抓取所有 4 态/4 行，无需轮询
- **Intl.DateTimeFormat 走 medium+short**：生成「2026年4月16日 00:57」无秒数，比 「YYYY-MM-DD HH:MM」更符合 zh-CN 用户阅读习惯（D-17）
- **重试 button 复用 useCallback(load, [])**：单一函数双入口（useEffect 自动触发 + ErrorCard onClick 手动触发），无重复逻辑
