   ```python

   ```

   - Body: `AdminBindRequest` (with `employee_id`)

   - Return 200 with updated `UserRead`

3. **Admin unbind endpoint** (per D-06, D-03) — In `backend/app/api/v1/users.py`, add:
   ```python
   @router.delete('/{user_id}/bind', response_model=UserRead)
   ```
   - Requires `require_roles('admin', 'hrbp', 'manager')`
   - Calls `UserAdminService.bind_employee(user_id, employee_id=None, operator=current_user)`
   - **After unbind**: increment the target user's `token_version` by 1 and commit. This invalidates all existing tokens for that user (per D-03).
   - Return 200 with updated `UserRead`

5. **Self-bind preview** (per D-06, D-02) — In `backend/app/api/v1/auth.py`, add:
   ```python
   @router.get('/self-bind/preview', response_model=SelfBindPreview)
   ```
   - Requires `get_current_user` (any authenticated user)
   - Query param: `id_card_no: str`
   - Temporarily set `current_user.id_card_no` to the provided value (do NOT commit), then call `IdentityBindingService.search_employee_for_user_by_identity(user=current_user)`
   - If no employee found: raise 404 "未找到匹配的员工信息"
   - If employee is already bound to another user: query bound user email and raise 409 with D-05 conflict message
   - If found and available: return `SelfBindPreview` with employee details
   - Restore original `id_card_no` after (use try/finally or just don't flush)

   - Set `current_user.id_card_no` to normalized value, then call `IdentityBindingService.auto_bind_user_and_employee(user=current_user)`
   - On ValueError (conflict): include bound email per D-05
   - If no matching employee found (auto_bind returns False): raise 404 "未找到匹配的员工信息"
   - Commit and return updated `UserRead`
