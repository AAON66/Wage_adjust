```python
class FeishuOAuthService:
    """飞书 OAuth2 授权码登录服务。"""

    FEISHU_AUTHORIZE_URL = 'https://accounts.feishu.cn/open-apis/authen/v1/authorize'
    FEISHU_TOKEN_URL = 'https://open.feishu.cn/open-apis/authen/v2/oauth/token'
    FEISHU_USER_INFO_URL = 'https://open.feishu.cn/open-apis/authen/v1/user_info'

    def __init__(self, db: Session, settings: Settings | None = None) -> None:
        self.db = db
        self._settings = settings or get_settings()
```

方法列表：

a. `generate_authorize_url(self, redis_client: redis.Redis) -> dict[str, str]`
   - 使用 `secrets.token_urlsafe(32)` 生成 state（per D-02）
   - `redis_client.setex(f'feishu_oauth_state:{state}', 300, '1')` 存入 Redis TTL 5 分钟
   - 用 `urllib.parse.urlencode` 拼接授权 URL，参数：client_id, response_type=code, redirect_uri, state
   - 返回 `{'authorize_url': url, 'state': state}`

b. `handle_callback(self, code: str, state: str, redis_client: redis.Redis) -> User`
   - 调用 `_validate_state(state, redis_client)` 校验 CSRF
   - 调用 `_check_code_replay(code, redis_client)` 检查 code 重放，若已使用抛出 HTTPException(400, '授权码已使用')
   - `redis_client.setex(f'feishu_oauth_code:{code}', 600, '1')` 记录已使用 code（per D-03, TTL 10 分钟）
   - 调用 `_exchange_code_for_token(code)` 获取 user_access_token
   - 调用 `_get_user_info(access_token)` 获取 open_id 和 employee_no
   - 调用 `_find_or_bind_user(open_id, employee_no)` 查找/绑定用户
   - 返回 User 对象

c. `_validate_state(self, state: str, redis_client: redis.Redis) -> None`
   - 检查 `feishu_oauth_state:{state}` 是否存在
   - 不存在时抛出 HTTPException(400, '无效的 state 参数，请重新发起授权')
   - 存在时立即删除（一次性使用，per D-02）

d. `_check_code_replay(self, code: str, redis_client: redis.Redis) -> None`
   - 检查 `feishu_oauth_code:{code}` 是否存在
   - 存在时抛出 HTTPException(400, '授权码已使用')

e. `_exchange_code_for_token(self, code: str) -> dict`
   - 使用 httpx.post 调用 FEISHU_TOKEN_URL（per D-01, RESEARCH code example）
   - JSON body: grant_type=authorization_code, client_id=settings.feishu_app_id, client_secret=settings.feishu_app_secret, code=code, redirect_uri=settings.feishu_redirect_uri
   - timeout=10
   - 错误码映射：20002->500 配置错误, 20003->400 已使用, 20004->400 已过期, 20010->403 未授权, 其他->502
   - 成功时返回完整 response JSON

f. `_get_user_info(self, access_token: str) -> dict`
   - httpx.get FEISHU_USER_INFO_URL with Bearer token header
   - 失败时抛出 HTTPException(502, '获取飞书用户信息失败')
   - 返回 data 字段

g. `_find_or_bind_user(self, open_id: str, employee_no: str) -> User`
   - Fast path（per D-08）：查 `User.feishu_open_id == open_id`，命中直接返回
   - Slow path：
     - 内联 _build_employee_map 逻辑（查询 Employee.employee_no, Employee.id 构建 map）
     - 调用 `FeishuService._lookup_employee(emp_map, employee_no)` 查找 employee_id
     - employee_id 为 None 时抛出 HTTPException(400, '工号未匹配，请联系管理员开通')（per D-09）
     - 查 `User.employee_id == employee_id`
     - User 不存在时抛出 HTTPException(400, '工号未匹配，请联系管理员开通')（per D-06）
     - 绑定 feishu_open_id：`user.feishu_open_id = open_id; self.db.flush()`
     - 返回 user

h. `require_redis(self) -> redis.Redis`
   - 获取 Redis 客户端，不可用时抛出 HTTPException(503, '认证服务暂不可用，请稍后重试')
   - 区别于 auth.py 的 graceful degradation（per RESEARCH Pitfall 1）

全部方法需要 `from __future__ import annotations` 开头。
导入列表：`import logging, secrets, urllib.parse`, `import httpx, redis as redis_lib`, `from fastapi import HTTPException`, `from sqlalchemy import select`, `from sqlalchemy.orm import Session`, 以及项目内部导入。

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 浏览器 -> /feishu/callback | 不可信输入：code 和 state 参数来自浏览器 URL 重定向 |
| 后端 -> 飞书 API | 外部 API 调用，响应可能被篡改（HTTPS 保护） |
| 后端 -> Redis | 内部信任，state/code 存取 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-03 | Spoofing | /feishu/callback | mitigate | state CSRF token 存 Redis TTL 5min，回调时校验后立即删除（FeishuOAuthService._validate_state） |
| T-26-04 | Repudiation | /feishu/callback | mitigate | authorization code 记录到 Redis TTL 10min 防重放（FeishuOAuthService._check_code_replay），飞书 API 自身也保证一次性使用 |
| T-26-05 | Tampering | redirect_uri | mitigate | redirect_uri 从 .env 固定配置，不接受请求参数动态值 |
| T-26-06 | Information Disclosure | user_access_token | mitigate | token 仅在 handle_callback 方法内使用后丢弃，不持久化不记录到日志 |
| T-26-07 | Denial of Service | /feishu/authorize | accept | 公开端点可被频繁调用创建 Redis state key，但 TTL 5min 自动过期，Redis 内存影响有限 |
| T-26-08 | Elevation of Privilege | /feishu/callback | mitigate | 不自动创建 User（per D-06），只绑定已存在的 User，不会产生无角色账号 |
| T-26-09 | Spoofing | Redis 不可用 | mitigate | Redis 不可用时返回 503 而非跳过校验（require_redis 方法），区别于登录 rate limiting 的 graceful degradation |
</threat_model>
