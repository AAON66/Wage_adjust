**Accent（`--color-primary` `#1456F0`）reserved-for 显式清单（10% 约束）：**
1. Tab 激活态下边框 `border-bottom: 2px solid var(--color-primary)` + 文字色（`EligibilityManagementPage.tsx:44` 既定行为，不改动）
2. 主 CTA「确认导入」`.action-primary` 背景色
3. Radio 选中态圆点填充 + 圆环描边
4. Preview `update` 计数卡片色族（标签文字 + 数值 + 背景）
5. diff 表格内「update」行左侧 3px 指示条
6. Drop zone 拖拽 hover 态边框（`ExcelImportPanel.tsx:119` 既定）
7. 「修改 Excel 后重新上传」次级 CTA hover 态
8. 进度条底色（上传中 / 落库中）

**不允许使用 accent 色：**
- 4 色计数卡片中的 insert / no_change / conflict（已分配独立 token）
- Radio 未选中态边框（用 `--color-border`）
- Modal 正文文字（用 `--color-ink`）
- diff 表格普通 cell 文字（用 `--color-ink` / `--color-steel`）
- 模板下载按钮（用 `.action-secondary`，边框 steel，文字 ink）

### Destructive 使用规则（`--color-danger`）

仅用于：
1. Preview `conflict` 计数卡片色族（数字 + 标签 + 背景）
2. diff 表格 `conflict` 行背景 + 左指示条
3. **replace 模式 inline 警告**（D-11）：`⚠` 符号 + 警告文字 + 左侧 3px 指示条均用 danger
4. **二次确认 modal 中「替换模式」「清空未填字段」关键词**：用 `--color-danger` 加粗标注；其他正文仍用 ink
5. **二次确认 modal 主 CTA「继续」按钮**：用 `.action-danger` 变体（红字白底红边），非 `.action-primary`（蓝底白字），与 Phase 21「拒绝重新上传」同风格
6. 409 冲突 toast 左侧图标色（toast 主体仍用 `--color-warning-bg` 橙底，与 `.action-danger` 的「红底白字」区分）

**本 phase 的破坏性语义清单（必须走二次确认）：**
- **replace 模式导入**：Radio 选中 replace → inline 警告 → 点「确认导入」弹 modal → 勾选「我已理解并确认」→ 点「继续」才落库
- **没有其他破坏性操作**：merge 模式（默认）无二次确认；取消 Preview 不需确认（未落库）；删除 Preview 文件由定时任务清理

**文案契约要点：**
1. **禁用口吻：** 所有禁用 tooltip 必须说明「为何禁用」 + 「该怎么办」，不仅显示「不可用」
2. **错误动作路径：** 所有错误态必须含「该怎么办」（重新上传 / 联系管理员 / 稍后重试）
3. **不暴露内部字段：** UI 显示「新增 / 更新 / 无变化 / 冲突」，不显示 `insert` / `update` / `no_change` / `conflict` 字面量；job status 走「预览待确认 / 落库中」而非 `previewing` / `processing`
4. **破坏性口吻分层：** inline 警告 = 提示（偏软）；二次确认 modal = 阻断（偏硬，带「不可自动恢复」强警示）
5. **合规边界：** 所有文案不泄露其他员工工号、薪资、绩效数字给当前 HR（员工端 Phase 35 才有此约束；HR 端导入看到完整 diff 是允许的）
