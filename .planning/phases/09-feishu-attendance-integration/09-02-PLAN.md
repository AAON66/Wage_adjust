   **CRITICAL: period 从源数据推导（addresses review HIGH #2）**
   - `_map_fields(record: dict, mapping: FieldMapping) -> dict | None`:
     - 按 mapping 将飞书字段名转换为系统字段名
     - 提取 `record_id` -> `feishu_record_id`
     - 提取 `last_modified_time` -> `source_modified_at`（unix 毫秒转 datetime）
     - **period 推导逻辑**（不使用 datetime.now()）:
       1. 如果 FieldMapping 中 `period` 字段有映射值（非空字符串），从飞书记录的对应字段提取月份
       2. 如果飞书字段值是数字类型（unix 毫秒时间戳），转换为 'YYYY-MM'
       3. 如果飞书字段值是字符串且匹配 'YYYY-MM' 格式，直接使用
       4. 如果飞书字段值是字符串且匹配 'YYYY-MM-DD' 或 'YYYY/MM/DD' 等，提取前 7 位 'YYYY-MM'
       5. 如果 FieldMapping 中没有 period 映射或提取失败: 使用 `source_modified_at`（飞书记录的 last_modified_time）转换为 'YYYY-MM' 作为 fallback
       6. 如果 `source_modified_at` 也为空，则 log warning 并跳过该记录（return None）
     - 类型强制转换（str->float/int），单条失败返回 None 并 log warning

   - `_upsert_records(mapped_records: list[dict]) -> tuple[int, int, list[str]]`:
     - 按 employee_no 查找 Employee 获取 employee_id
     - upsert 策略: 按 `feishu_record_id` 查找已有记录（唯一索引）。找到则更新字段值。未找到则尝试 INSERT。INSERT 时如果 `(employee_id, period)` 冲突，则更新已有行。
     - 设置 `synced_at = datetime.now(UTC)`
     - period 值来自 _map_fields 输出，**不再接受 current_period 参数**
     - 返回 (matched, unmatched, unmatched_nos)

   **CRITICAL: 原子并发防护（addresses review HIGH #1）**
   - `_claim_sync(mode: str) -> FeishuSyncLog | None`:
     - 原子操作: 在一个事务中:
       1. 查询最近 1 小时内是否有 status='running' 的 FeishuSyncLog（使用 SELECT FOR UPDATE 或 with_for_update() 如果 DB 支持）
       2. 如果有，返回 None（拒绝）
       3. 如果没有，INSERT 新的 FeishuSyncLog(mode=mode, status='running', started_at=now)
       4. flush 并返回新创建的 log 记录
     - 如果 DB 不支持 FOR UPDATE（如 SQLite 开发环境），降级为: INSERT + 捕获 IntegrityError
     - 对于 SQLite 兼容性:
       1. 先 INSERT FeishuSyncLog(status='running')
       2. 然后查询 1 小时内 status='running' 的记录数
       3. 如果 > 1，说明有竞争，将自己的记录改为 status='failed'，返回 None
     - 超过 1 小时仍为 running 的记录视为僵尸，先将其 status 改为 'failed'(error_message='同步超时，自动标记失败') 再 claim

   - `sync_attendance(mode: str = 'incremental') -> SyncResultResponse`:
     - 调用 `_claim_sync(mode)` 获取 sync_log
     - 如果返回 None: 返回 SyncResultResponse(status='failed', error_message='同步任务正在进行中，请稍后再试')
     - 获取活跃配置，解密 Secret，获取 token
     - **增量模式水印策略**:

       3. 将水印时间传给 `_fetch_all_records` 的 `since_timestamp` 参数

     - **全量模式**: `since_timestamp=None`，拉取全部记录 + upsert
     - 调用 _fetch_all_records + _upsert_records
     - 更新 sync_log: status='success'/'failed'，填写 finished_at, total_records, matched_records, unmatched_records
     - 返回 SyncResultResponse

       - 全部失败: log error + 返回最终失败的 SyncResultResponse

   ```python
   from apscheduler.schedulers.asyncio import AsyncIOScheduler
   from apscheduler.triggers.cron import CronTrigger
   import logging
   from backend.app.core.database import SessionLocal

   logger = logging.getLogger(__name__)
   scheduler = AsyncIOScheduler()

   def _run_incremental_sync():
       """定时同步 job — 在独立 DB Session 中执行"""
       db = SessionLocal()
       try:
           from backend.app.services.feishu_service import FeishuService
           service = FeishuService(db)
           result = service.sync_with_retry(mode='incremental')
           db.commit()
           logger.info('定时考勤同步完成: %s', result)
       except Exception:
           db.rollback()
           logger.exception('定时考勤同步失败')
       finally:
           db.close()
   ```

   **CRITICAL: 无配置时跳过（addresses review MEDIUM #4）**
   ```python
   def start_scheduler(sync_hour: int = 6, sync_minute: int = 0):
       db = SessionLocal()
       try:
           from backend.app.services.feishu_service import FeishuService
           config = FeishuService(db).get_config()
           if not config:
               logger.info('未检测到飞书配置，跳过定时同步调度器注册')
               return  # 不注册 job，不启动 scheduler
           sync_hour = config.sync_hour
           sync_minute = config.sync_minute
       finally:
           db.close()

       scheduler.add_job(
           _run_incremental_sync,
           trigger=CronTrigger(hour=sync_hour, minute=sync_minute),
           id='feishu_attendance_sync',
           replace_existing=True,
       )
       scheduler.start()
       logger.info('考勤同步调度器已启动: 每天 %02d:%02d', sync_hour, sync_minute)

   def stop_scheduler():
       if scheduler.running:
           scheduler.shutdown(wait=False)

   def reschedule(sync_hour: int, sync_minute: int):
       """配置更新后重新调度。如果调度器未运行，先启动。"""
       if not scheduler.running:
           scheduler.add_job(
               _run_incremental_sync,
               trigger=CronTrigger(hour=sync_hour, minute=sync_minute),
               id='feishu_attendance_sync',
               replace_existing=True,
           )
           scheduler.start()
       else:
           scheduler.reschedule_job(
               'feishu_attendance_sync',
               trigger=CronTrigger(hour=sync_hour, minute=sync_minute),
           )
       logger.info('考勤同步已重新调度: 每天 %02d:%02d', sync_hour, sync_minute)
   ```
