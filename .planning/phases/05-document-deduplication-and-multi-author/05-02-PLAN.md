**SUB-02 贡献比例规则 / Contribution percentage rules:**
owner_pct = 100 - sum(contributor_pcts)。系统自动计算 owner 份额，用户只输入每个协作者的百分比。
如果 sum(contributor_pcts) >= 100，拒绝请求。用户不需要也不应该手动输入 owner_pct。
owner_pct = 100 - sum(contributor_pcts). System auto-computes owner share; user only inputs each contributor's percentage.
If sum(contributor_pcts) >= 100, reject the request. User does NOT manually input owner_pct.

  <action>
    在 FileService 中添加以下方法和修改:

    1. **_compute_hash(content: bytes) -> str**:
       ```python
       import hashlib
       def _compute_hash(self, content: bytes) -> str:
           return hashlib.sha256(content).hexdigest()
       ```

    2. **_check_duplicate(file_name: str, content_hash: str, *, exclude_file_id: str | None = None) -> UploadedFile | None** (per D-02 全局去重):
       - 查询所有 UploadedFile 中 file_name 和 content_hash 都匹配的记录
       - **不过滤 employee_id, 不过滤 submission_id**（D-02 锁定决策: 全局去重，不同人不能提交完全相同文件。RESEARCH.md 建议 per-employee 去重，但被 CONTEXT.md D-02 覆盖）
       - 查询示例: `session.query(UploadedFile).filter(UploadedFile.file_name == file_name, UploadedFile.content_hash == content_hash)`
       - 如果 exclude_file_id 不为 None，额外添加 `.filter(UploadedFile.id != exclude_file_id)`（用于 replace_file）
       - 返回第一条匹配记录或 None

    3. **_validate_contributors(contributors: list[ContributorInput]) -> None**:
       - 计算 total_contributor_pct = sum(c.contribution_pct for c in contributors)
       - 验证 total_contributor_pct < 100（上传者保留剩余比例，即 owner_pct = 100 - total_contributor_pct）
       - 验证每个 contribution_pct > 0
       - 验证无重复 employee_id
       - 失败时 raise ValueError 含中文提示

    4. **_save_contributors(file_record: UploadedFile, submission: EmployeeSubmission, contributors: list[ContributorInput]) -> None**:
       - 为每个 contributor 查找或创建 EmployeeSubmission（同 cycle）
       - 创建 ProjectContributor 记录（status="accepted"）
       - 更新 file_record.owner_contribution_pct = 100 - sum(pcts)

    5. **修改 _create_file_record** -- 接受新参数 `content_hash: str`，存入模型字段

    6. **修改 upload_files** -- 签名新增 `contributors: list[ContributorInput] | None = None`:
       - 读取文件内容后调用 _compute_hash
       - 调用 _check_duplicate，如有重复 raise ValueError 含已有记录信息（per D-03: "此文件已由 XXX 在 YYYY-MM-DD 提交"）
       - _create_file_record 传入 content_hash
       - 如有 contributors，调用 _validate_contributors 和 _save_contributors
       - D-04: 去重在上传前完成

    7. **修改 import_github_file** -- 下载后计算 hash，调用 _check_duplicate

    8. **修改 replace_file** -- 计算新文件 hash，调用 _check_duplicate(exclude_file_id=file_id)，更新 content_hash

    9. **修改 list_files** -- 合并查询:
       - 原始查询：submission 直接拥有的文件
       - 新增查询：通过 ProjectContributor 关联的共享文件（标记 is_shared=True）
       - 返回合并列表，共享文件附加 metadata 标识

    10. **新增 dispute_contribution(contributor_id: str, disputer_employee_id: str) -> ProjectContributor** (per D-06):
        - 查找 ProjectContributor by id
        - 验证 disputer_employee_id 是该 contributor 记录对应的员工（通过 submission -> employee 关联验证，只有被分配比例的人可以异议）
        - 将 status 从 "accepted" 改为 "disputed"
        - 如果 status 已经是 "disputed" 或 "resolved"，raise ValueError("状态不允许异议")
        - 返回更新后的记录

    11. **新增 resolve_dispute(contributor_id: str, *, resolution: str, resolver_employee_id: str, new_pct: float | None = None) -> ProjectContributor** (per D-06):
        - resolution 可以是 "all_confirmed" 或 "manager_override"
        - "all_confirmed": 验证所有相关 contributor 都已确认（可通过单独的确认端点记录），将 status 改为 "resolved"
        - "manager_override": 验证 resolver 有管理者权限（通过 employee.role 或 require_roles 检查），直接将 status 改为 "resolved"，如有 new_pct 则更新贡献比例（同时重新计算 owner_pct = 100 - sum(all_contributor_pcts)）
        - 返回更新后的记录
