  <action>
    在 FileService 中添加以下方法和修改:

    1. **_compute_hash(content: bytes) -> str**:
       ```python
       import hashlib
       def _compute_hash(self, content: bytes) -> str:
           return hashlib.sha256(content).hexdigest()
       ```

    2. **_check_duplicate(file_name: str, content_hash: str, *, exclude_file_id: str | None = None) -> UploadedFile | None** (per D-02 全局去重):
       - 查询所有 UploadedFile 中 file_name 和 content_hash 都匹配的记录
       - 不过滤 employee_id（D-02: 全局去重，不同人不能提交完全相同文件）
       - 如果 exclude_file_id 不为 None，排除该记录（用于 replace_file）
       - 返回第一条匹配记录或 None

    3. **_validate_contributors(contributors: list[ContributorInput]) -> None**:
       - 验证 sum(pct) < 100（上传者保留剩余比例）
       - 验证每个 contribution_pct > 0
       - 验证无重复 employee_id
       - 失败时 raise ValueError 含中文提示

    4. **_save_contributors(file_record: UploadedFile, submission: EmployeeSubmission, contributors: list[ContributorInput]) -> None**:
       - 为每个 contributor 查找或创建 EmployeeSubmission（同 cycle）
       - 创建 ProjectContributor 记录
       - 更新 file_record.owner_contribution_pct = 100 - sum(pcts)

    5. **修改 _create_file_record** — 接受新参数 `content_hash: str`，存入模型字段

    6. **修改 upload_files** — 签名新增 `contributors: list[ContributorInput] | None = None`:
       - 读取文件内容后调用 _compute_hash
       - 调用 _check_duplicate，如有重复 raise ValueError 含已有记录信息（per D-03: "此文件已由 XXX 在 YYYY-MM-DD 提交"）
       - _create_file_record 传入 content_hash
       - 如有 contributors，调用 _validate_contributors 和 _save_contributors
       - D-04: 去重在上传前完成

    7. **修改 import_github_file** — 下载后计算 hash，调用 _check_duplicate

    8. **修改 replace_file** — 计算新文件 hash，调用 _check_duplicate(exclude_file_id=file_id)，更新 content_hash

    9. **修改 list_files** — 合并查询:
       - 原始查询：submission 直接拥有的文件
       - 新增查询：通过 ProjectContributor 关联的共享文件（标记 is_shared=True）
       - 返回合并列表，共享文件附加 metadata 标识
  </action>
