## Phase 2 Specific Constraints

### EVAL-06: Fallback Banner Placement

The `FallbackBanner` must render:
1. Inside the `.surface` card that contains the evaluation content
2. Above the dimension score section and review panel
3. At all flow stages where evaluation data is visible (evaluated, reviewing, calibrated, approved)
4. Never in the "collecting" or "submitted" stages (no evaluation yet)

### EVAL-07: Dimension Summary Panel Placement

The `DimensionSummaryPanel` (read-only) must render:
1. As a separate panel from `DimensionScoreEditor` (edit mode)
2. Visible to all roles (employee, manager, hrbp, admin) when `status` is `evaluated` or later
3. `DimensionScoreEditor` remains visible only to manager/hrbp/admin in `reviewing` stage

### EVAL-05: Prompt Hash

The prompt hash (`sha256` of serialized messages) is an audit detail, not a primary UI element:
- Render inside a `<details>` accordion on each dimension row
- Do not show by default ŌĆö reduces visual noise for non-audit use cases
- Always render when present; never show "generating..." placeholder

### Score Display ŌĆö 5 Dimension Rows

Each dimension row in `DimensionSummaryPanel` must display:

| Element | Implementation |
|---------|---------------|
| Dimension code | `11px uppercase tracking-wide --color-primary` (eyebrow pattern) |
| Dimension label | `15px weight 600 --color-ink` (section-title pattern) |
| Weight percentage | `12px --color-steel` ŌĆö e.g., "µØāķćŹ 30%" |
| AI score | `26px weight 700 --color-ink` ŌĆö e.g., "78" |
| LLM rationale text | `13px --color-steel line-height 1.65` below score |
| Prompt hash | Inside `<details>`, `11px monospace --color-placeholder` |
