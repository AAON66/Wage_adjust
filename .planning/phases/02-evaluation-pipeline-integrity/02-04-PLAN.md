<objective>
Fix the scale normalization bug in the evaluation engine, wire prompt_hash and used_fallback
through to storage (including re-evaluation row cleanup), ensure sanitization is explicitly
called before evidence text enters LLM prompts, ensure the API surfaces used_fallback, and
extend prompt_safety.py with English and Unicode homoglyph injection patterns.

Purpose: EVAL-04 prevents silent score inflation when a 5-point overall_score appears alongside
100-point dimensions. EVAL-07 (backend portion) ensures used_fallback is correctly written and
returned by the API. EVAL-08 extends injection defense to English-language manipulation attempts.

Two HIGH issues from cross-AI review addressed here:
1. Re-evaluation semantics: old DimensionScore rows must be DELETED before inserting new ones
   to prevent score accumulation / audit drift on re-runs.
2. Sanitization wiring: scan_for_prompt_manipulation must be called on evidence text and
   OCR-derived text at the exact point where it is embedded in LLM prompt messages, not just
   detected in isolation.

- evaluation_service.py: fixed _normalize_llm_evaluation_payload (>=3 dims guard, ambiguity
  discard, 100.0 clamp); generate_evaluation sets used_fallback and prompt_hash on every write;
  re-evaluation path deletes old DimensionScore rows in the same transaction; evidence text
  sanitized before prompt construction
- evaluations.py: serialize_evaluation includes used_fallback=evaluation.used_fallback
- prompt_safety.py: 3 new patterns (english_score_manipulation, english_instruction_override,
  unicode_homoglyph) appended to PROMPT_MANIPULATION_PATTERNS
