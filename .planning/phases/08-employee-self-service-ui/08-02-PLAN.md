注意：审批历史需要 recommendation_id。获取路径是先拿到 salaryRecommendation 的 id，再查询审批历史。由于 salary 和 approval 有依赖关系，实际顺序为：
1. 获取 evaluation（已有 submission_id）
2. 获取 salary recommendation（用 evaluation.id）
3. 获取 approval history（用 salary.id 即 recommendation_id）

如果 salary 不存在，approval history 也无法获取，两者都设为空。


**4. 步骤条映射函数**（per D-01, 研究报告步骤映射逻辑）

在组件函数内新增辅助函数：
```typescript
function resolveCurrentStep(
  submissionStatus: string,
  salaryStatus: string | null,
  approvals: ApprovalRecord[],
): number {
  // 步骤 3: 已完成
  if (salaryStatus === 'approved') return 3;

  // 从 approval records 判断
  const currentPending = approvals.find(
    (r) => r.is_current_step && r.decision === 'pending',
  );
  if (currentPending) {
    // 使用 step_order: 1 → 主管审核(step 1), 2 → HR 审核(step 2)
    if (currentPending.step_order === 1) return 1;
    if (currentPending.step_order >= 2) return 2;
  }

  // 默认: 已提交
  return 0;
}
```
