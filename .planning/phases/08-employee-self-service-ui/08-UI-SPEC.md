## 交互合约 / Interaction Contracts

### EvaluationStepBar 步骤条

- **结构**：横向 4 步骤，圆形节点 + 连线 + 文字标签
- **步骤定义**：`已提交` → `主管审核` → `HR 审核` → `已完成`
- **节点**：28px 圆形，居中数字或勾号
- **连线**：节点间水平线，高度 2px
- **响应式**：桌面端横向排列；768px 以下仍保持横向但标签改为缩写或换行
- **Props**：`currentStep: number`（0-3，0=已提交，3=已完成）
- **无交互**：纯展示组件，不可点击

### DimensionRadarChart 雷达图

- **结构**：ECharts `radar` 类型图表
- **数据**：5 个维度，对应 5 个轴
- **轴标签**：中文维度名（AI工具掌握度、AI应用深度、AI学习能力、AI分享贡献、AI成果转化）
- **数值范围**：0-100
- **容器高度**：320px
- **交互**：hover 显示 tooltip（维度名 + 得分 + 权重）
- **空状态**：不渲染（由父组件根据状态控制显隐）
- **Props**：`scores: Array<{ dimension_code: string; raw_score: number; weight: number }>`

### DimensionCard 维度卡片

- **结构**：`surface-subtle` 卡片，内含维度名、得分、权重、LLM 说明
- **布局**：卡片内顶部一行放维度名和得分/权重，下方放说明文字
- **得分格式**：整数（如 `82`），满分 100
- **权重格式**：百分比（如 `30%`）
- **说明文字**：`text-sm text-steel`，最大高度不限（完整展示 LLM 说明）
- **容器布局**：5 张卡片使用 `grid gap-4 md:grid-cols-2 xl:grid-cols-3` 排列
- **Props**：`dimensionCode: string; rawScore: number; weight: number; rationale: string`

### SalaryResultCard 调薪结果卡片

- **结构**：`surface` 卡片，居中展示调薪百分比
- **百分比格式**：`+12.00%`（正数带加号，保留 2 位小数）
- **百分比字号**：26px, weight 600, `--color-primary` 色
- **副标题**：`text-sm text-steel`，文案见文案合约
- **仅在 approved 状态渲染**
- **Props**：`adjustmentRatio: number`（0.12 表示 12%）

## 状态到步骤映射 / Status-to-Step Mapping

| submission.status / approval 状态 | 步骤条 currentStep |
|----------------------------------|-------------------|
| `submitted` | 0（已提交，高亮） |
| `evaluated` / `reviewing` — 待主管审核 | 1（主管审核，高亮） |
| `reviewing` — 待 HR 审核 | 2（HR 审核，高亮） |
| `approved` / `published` | 3（已完成，全部勾选） |

具体映射逻辑需结合 approval_records 中当前待处理步骤判断。如无 approval 记录，则默认 step 0。
