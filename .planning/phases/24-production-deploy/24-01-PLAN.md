将现有 Dockerfile 升级为生产级配置（per D-01, D-11, DEPLOY-03）：

```dockerfile
FROM python:3.9-slim AS backend

WORKDIR /app

# 系统依赖（psycopg2-binary 需要 libpq）
RUN apt-get update && apt-get install -y --no-install-recommends \
    libpq-dev \
    && rm -rf /var/lib/apt/lists/*

# 先复制依赖文件利用 Docker 缓存
COPY requirements.txt requirements-prod.txt ./
RUN pip install --no-cache-dir -r requirements-prod.txt

# 复制后端代码
COPY backend/ backend/
COPY alembic/ alembic/
COPY alembic.ini ./

# 创建上传目录
RUN mkdir -p uploads

EXPOSE 8011

# 生产环境使用 gunicorn + uvicorn worker（per D-11）
# --workers 4: 默认 4 个 worker（Claude's Discretion）
# --timeout 120: AI 调用可能较慢，设 120s 超时
# --access-logfile -: 输出到 stdout 便于 docker logs 查看
CMD ["gunicorn", "backend.app.main:app", \
     "--worker-class", "uvicorn.workers.UvicornWorker", \
     "--workers", "4", \
     "--bind", "0.0.0.0:8011", \
     "--timeout", "120", \
     "--access-logfile", "-", \
     "--error-logfile", "-"]
```

关键设计点：
- 使用 `python:3.9-slim` 保持与项目 Python 版本一致（Phase 18 决定）
- 安装 `libpq-dev` 以支持 psycopg2（per D-03 生产用 PostgreSQL）
- 分层 COPY：先依赖文件后代码，利用 Docker build cache
- 使用 `requirements-prod.txt`（包含 gunicorn）
- 只复制 `backend/`、`alembic/`、`alembic.ini`，不复制 frontend（前端在独立容器）
- CMD 使用 JSON 数组格式确保信号传播
- 日志输出到 stdout/stderr 便于 `docker logs` 查看
