## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, metric-label to metric-value gap |
| sm | 8px | Compact element spacing, pill internal padding |
| md | 16px | Default element spacing, card internal padding, grid gaps |
| lg | 20px | Section padding (matches existing `app-main` padding) |
| xl | 24px | Card padding (matches existing `surface px-6 py-6`) |
| 2xl | 32px | Page bottom padding |

Exceptions: `lg` (20px) is not in the standard 8-point set. Retained for `app-main` padding compatibility -- the existing `.app-main` container uses 20px padding throughout the application and this phase must not alter that inherited value.

Source: Existing `app-main` (20px/24px padding), `metric-tile` (16px/18px padding), `surface` card patterns.

## Typography

| Role | Size | Weight | Line Height | CSS Class / Usage |
|------|------|--------|-------------|-------------------|
| Body | 13.5px | 400 | 1.5 | Default body text, table cells, descriptions |
| Label | 12px | 400 | 1.5 | `metric-label`, status labels, secondary info (differentiated by size, not weight) |
| Section Title | 15px | 600 | 1.4 | `section-title`, card headings |
| Display Number | 26px | 600 | 1.1 | `metric-value`, featured salary percentage |

Weights: 2 (400 regular, 600 semibold). Label uses size differentiation (12px vs 13.5px body) instead of a third weight.

Source: Existing `index.css` component classes (`metric-label`, `metric-value`, `section-title`, body defaults).

## Layout Contract

### Summary Layer (default visible)

```
+------------------------------------------------------------------+
| [eyebrow: 调薪建议]                              [状态: status]  |
| [section-title: 建议结果快照]                                     |
+------------------------------------------------------------------+
| [AttendanceKpiCard]  [EligibilityBadge]  [AI Score Card]         |
|   (surface-subtle)     (status-pill +      (surface-subtle)      |
|                         inline expand)                            |
+------------------------------------------------------------------+
| [Featured: 最终调薪比例]                                          |
|   metric-value 26px, color primary                                |
+------------------------------------------------------------------+
| [人工调整 btn]  [提交审批 btn]  [查看审批中心 link]               |
+------------------------------------------------------------------+
| [展开详情 btn - action-secondary, full-width]                     |
+------------------------------------------------------------------+
```

- Three indicator cards use `grid gap-3 md:grid-cols-3`
- Featured ratio uses `metric-value` class with `color: var(--color-primary)`
- Action buttons pinned at summary bottom (D-12, D-14)
- "展开详情" button below action buttons, full container width

### Detail Layer (toggle visible, D-05/D-06/D-07)

```
+------------------------------------------------------------------+
| [Dimension Score Table - 5 rows]                                  |
|   table-lite: 维度名 | 得分 | 权重 | 加权分                      |
+------------------------------------------------------------------+
| [建议涨幅 / AI 系数 / 认证加成 - 3-col grid]                     |
+------------------------------------------------------------------+
| [联动预览 panel - if available]                                   |
+------------------------------------------------------------------+
| [建议说明 - details/summary]                                      |
+------------------------------------------------------------------+
| [SalaryHistoryPanel]                                              |
+------------------------------------------------------------------+
```

- Detail layer renders below summary layer inside the same `.surface` card
- Separated from summary by a `<hr className="divider" />`
- Each sub-section uses existing spacing: `mt-5` (20px gap)

### Eligibility Badge Inline Expand (D-10, D-11)

```
[合格]  <-- status-pill, clickable
  v expanded:
  +----------------------------------------------+
  | [check] 入职满 6 个月          通过           |
  | [check] 距上次调薪满 6 个月    通过           |
  | [cross] 年度绩效 C 级以下      不合格         |
  | [check] 非法定假期未超 30 天   通过           |
  +----------------------------------------------+
```

- Expand area uses `surface-subtle` background with `mt-2` offset below badge
- Each rule row: flex layout, icon (inline SVG check/cross) + rule name + status text
- Rule row height: 32px (8px vertical padding per row)
- Check icon color: `var(--color-success)` / Cross icon color: `var(--color-danger)`

## Interaction Contract

### Expand/Collapse Detail (D-05, D-06, D-07)

| Trigger | Action | State |
|---------|--------|-------|
| Click "展开详情" | Show detail layer, button label changes to "收起" | `isDetailExpanded: true` |
| Click "收起" | Hide detail layer, button label changes to "展开详情" | `isDetailExpanded: false` |

- Controlled by React `useState<boolean>(false)` (D-07)
- Animation: `animate-fade-soft` (0.18s ease opacity fade) on detail layer mount
- No scroll-to behavior on expand

### Eligibility Badge Expand (D-10, D-11)

| Trigger | Action | State |
|---------|--------|-------|
| Click badge pill | Toggle inline rule list below badge | `isBadgeExpanded: boolean` |

- Inline expand, no modal, no navigation (D-11)
- Animation: `animate-fade-soft` on rule list mount

### Human Adjustment (D-12, D-13)

| Trigger | Action | State |
|---------|--------|-------|
| Click "人工调整" | Opens salary editor (reuses `isSalaryEditorOpen`) | Existing state |

- Button always visible in summary layer (D-12, D-14)
- Existing implementation reused without change

### Empty State (D-17, D-18)

| Condition | Display |
|-----------|---------|
| No salary recommendation | Summary header visible, indicator cards for eligibility/AI score if available, salary area shows dashed-border empty state with "还未生成调薪建议" text + "生成调薪建议" button |
| No eligibility data | Badge shows placeholder text "资格待检" in `surface-subtle` style with `color: var(--color-placeholder)` |
