|--------|-------|-------------|------------------|

**Accent（`--color-primary` `#1456F0`）reserved-for 显式清单（10% 约束）：**
1. 左侧导航菜单「同步日志」激活态背景 + 图标色（沿用 `AppShell.tsx` 既定 `.nav-link-active` 模式）
2. Tab 激活态下边框 `border-bottom: 2px solid var(--color-primary)` + 文字色
3. 抽屉标题栏左侧 4px 彩条
4. 表格内「刷新状态」/「触发同步」次级 CTA hover 态（`.chip-button:hover` 颜色）
5. 数值超链接式「下载未匹配工号 CSV」按钮文字色
6. 运行中状态 `running` 的 spinner + badge
7. 「查看详情」抽屉触发按钮文字色（行内链接）

**不允许使用 accent 色：**
- 表格所有普通行（ink + steel）
- badge 中的 5 类计数（用 5 类专属 token，不用 primary）
- 分页按钮（用 steel + border）
- 空状态图标 / 错误提示文字（用 steel / danger）

### Destructive 使用规则

`--color-danger` 仅用于：
1. `status=failed` 徽章
2. `failed_count` 五色 badge 段数值
3. 抽屉内 `error_message` 文字色
4. 若未来出现删除日志功能（本期不做），按钮用 danger

**本 phase 无删除/破坏性操作** — Phase 31 只读 + 诊断，不提供日志删除 / 取消同步等破坏性入口（见 Copywriting Contract 「Destructive confirmation」行）。

| Error state（CSV 下载失败） | 未匹配工号导出失败：{error}。请稍后重试。 |

| 抽屉标题 | 同步明细 #{log_id_short} |
| 抽屉内 `leading_zero_fallback_count > 0` 黄字 | {N} 条记录通过前导零容忍匹配成功，建议排查飞书源数据格式 |
| 抽屉内未匹配工号列表空状态 | 本次同步无未匹配工号 |
| 抽屉内 `error_message` 标签 | 错误信息 |
| 时间戳格式 | 2026-04-21 14:32（不含秒，与 `SyncStatusCard.formatSyncTime` 一致） |
| Partial 状态说明（抽屉脚注） | 同步完成但存在未匹配 / 映射失败 / 写库失败条目，请点击详情查看根因。 |
| Destructive confirmation | 不适用（本 phase 不提供删除/取消/破坏性操作） |

**文案契约要点：**
1. **禁用口吻：** 禁用按钮的 tooltip 必须说明「为何禁用」，不仅显示「不可用」
2. **错误动作路径：** 所有错误态必须含「该怎么办」（刷新 / 联系管理员 / 稍后重试）
3. **数字与名词搭配：** "N 条" 不省略单位；"N 次" 仅用于触发次数
4. **409 与 partial 的语气差异：** 409 是即时阻断（偏提醒），partial 是事后诊断（偏描述）
5. **不暴露内部字段名：** UI 显示「未匹配」/「映射失败」，不显示 `mapping_failed_count` 或 `unmatched_employee_nos` 字面量

## Accessibility Baseline

1. **颜色对比度：** 所有 badge 文字色与 bg 组合必须 ≥ 4.5:1。现有 `index.css` token 已验证（绿 `#00B42A` on `#E8FFEA` = 4.76:1，橙 `#FF7D00` on `#FFF3E8` = 4.51:1，红 `#F53F3F` on `#FFECE8` = 4.71:1，蓝 `#1456F0` on `#EBF0FE` = 6.82:1）。**新增的紫色 `#722ED1` on `#F5EBFA` = 7.13:1，PASS**。
2. **Focus 态：** 全站 `*:focus-visible { outline: 2px solid var(--color-primary); outline-offset: 2px }` 继承即可。
3. **语义 HTML：** 表格用 `<table><thead><tbody>`；Tab 用 `role="tablist" / role="tab"`；抽屉用 `role="dialog" aria-modal="true" aria-labelledby`。
4. **Loading / Live region：** 列表 refetch 时 `aria-live="polite"` 宣告「已更新 N 条记录」。
5. **CSV 下载错误：** 以 toast 形式宣告（`role="alert"`），不仅靠颜色。

## New Design Tokens（本 phase 引入）

仅新增 2 个 CSS 变量，位于 `frontend/src/index.css` `:root` 段末尾（在 `--color-info-bg` 之后）：

/* Phase 31: mapping_failed 专属紫色 */
--color-violet: #722ED1;
--color-violet-bg: #F5EBFA;
