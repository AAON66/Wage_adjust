         - `rule_code: str` -- one of: TENURE, ADJUSTMENT_INTERVAL, PERFORMANCE, LEAVE
         - `rule_label: str` -- Chinese label for display
         - `status: str` -- one of: 'eligible', 'ineligible', 'data_missing'

       - `@dataclass class EligibilityResult`:
         - `overall_status: str` -- one of: 'eligible', 'ineligible', 'pending'

         - Docstring: "Terminology mapping (ELIG-08/D-10): per-rule 'data_missing' maps to overall 'pending' when no rule is 'ineligible'"
       - `class EligibilityEngine`:
         - `__init__(self, thresholds: EligibilityThresholds | None = None)` -- defaults to EligibilityThresholds()
         - `_month_diff(self, earlier: date, later: date) -> int` -- formula: `(later.year - earlier.year) * 12 + (later.month - earlier.month)`. This gives integer month granularity. Exactly 6 calendar months apart = returns 6 = eligible. Document: "Month calculation uses year*12+month arithmetic. Day-of-month is ignored. Example: 2026-01-15 to 2026-07-15 = 6 months; 2026-01-31 to 2026-07-01 = 6 months."
         - `check_tenure(self, hire_date: date | None, reference_date: date) -> RuleResult`:
           - if hire_date is None: return data_missing with detail '入职日期未录入'
           - months = self._month_diff(hire_date, reference_date)
           - if months >= threshold: eligible with detail f'已入职 {months} 个月'
           - else: ineligible with detail f'入职仅 {months} 个月，需满 {self.thresholds.min_tenure_months} 个月'
           - rule_code='TENURE', rule_label='入职时长'
         - `check_adjustment_interval(self, last_adjustment_date: date | None, reference_date: date) -> RuleResult`:
           - if last_adjustment_date is None: return data_missing with detail '无调薪记录' -- addresses review concern: no-history = data_missing, NOT ineligible
           - months = self._month_diff(last_adjustment_date, reference_date)
           - if months >= threshold: eligible; else ineligible
           - rule_code='ADJUSTMENT_INTERVAL', rule_label='调薪间隔'
         - `check_performance(self, grade: str | None) -> RuleResult`:
           - if grade is None: return data_missing with detail '绩效数据未导入'
           - Normalize: grade_upper = grade.strip().upper()
           - if grade_upper not in GRADE_ORDER: return data_missing with detail f'绩效等级 "{grade}" 无法识别'
           - if grade_upper in self.thresholds.performance_fail_grades: ineligible with detail f'绩效等级为 {grade_upper}，低于合格线'
           - else: eligible with detail f'绩效等级为 {grade_upper}'
           - rule_code='PERFORMANCE', rule_label='绩效等级'
         - `check_leave(self, non_statutory_leave_days: float | None) -> RuleResult`:
           - if non_statutory_leave_days is None: return data_missing with detail '非法定假期数据未导入'
           - if non_statutory_leave_days > threshold: ineligible (strictly greater than; exactly 30 = eligible)
           - else: eligible
           - rule_code='LEAVE', rule_label='非法定假期'
         - `evaluate(self, *, hire_date: date | None, last_adjustment_date: date | None, performance_grade: str | None, non_statutory_leave_days: float | None, reference_date: date) -> EligibilityResult`:
           - calls all 4 check methods
           - overall_status logic (per D-10, addresses ELIG-08):
             - if ANY rule has status 'ineligible' -> overall = 'ineligible'
             - if ALL rules have status 'eligible' -> overall = 'eligible'
             - otherwise (no ineligible but some data_missing) -> overall = 'pending'
           - returns EligibilityResult
       - NO database imports, NO session dependencies. Pure computation only.
