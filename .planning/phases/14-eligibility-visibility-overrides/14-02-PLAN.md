From backend API (created in Plan 01):
```
GET  /api/v1/eligibility/batch?department=&status=&rule=&job_family=&job_level=&page=1&page_size=50&year=
  -> { items: EligibilityBatchItem[], total: number, page: number, page_size: number }

GET  /api/v1/eligibility/batch/export?department=&status=&rule=&job_family=&job_level=&year=
  -> Excel file download (full filtered dataset, max 5000 rows)

POST /api/v1/eligibility/overrides  (manager/hrbp ONLY -- admin gets 403)
  body: { employee_id, override_rules: string[], reason, year?, reference_date? }
  -> OverrideRequestRead (201)
  Backend validates: requester access, employee ineligible, rules actually failing

GET  /api/v1/eligibility/overrides?status=&page=1&page_size=50
  -> { items: OverrideRequestRead[], total, page, page_size }

POST /api/v1/eligibility/overrides/{id}/decide  (role-step binding enforced by backend)
  body: { decision: 'approve'|'reject', comment?: string }
  -> OverrideRequestRead

GET  /api/v1/eligibility/overrides/{id}
  -> OverrideRequestRead
```
