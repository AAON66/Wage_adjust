<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shared employee API payload -> multiple frontend consumers | company 字段进入 shared contract 后，任何页面都可能意外开始渲染它 |
| admin form state -> employee PATCH/POST payload | 若 types 和表单状态不同步，company 可能无法提交或被提交成错误值 |
| detail page rendering -> list page visibility requirement | 同一 employee contract 同时服务 detail 与 list，必须用 UI guardrail 保持边界 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-05 | Information Disclosure | `frontend/src/pages/Employees.tsx` | mitigate | 保持列表页字段集合不变，并用 grep-verifiable acceptance criteria 明确禁止 `employee.company` / `所属公司` 出现在该页 |
| T-20-06 | Integrity | `frontend/src/components/employee/EmployeeArchiveManager.tsx` + `frontend/src/types/api.ts` | mitigate | form state、payload types、edit 回填同步增加 `company`，避免提交时丢字段或类型不匹配 |
| T-20-07 | Spoofing / Misrepresentation | `frontend/src/pages/EvaluationDetail.tsx` | mitigate | 详情页仅显示来自 `fetchEmployee()` 的 `employee.company`，不拼接派生文本或其他来源，空值显示 `未设置` |
