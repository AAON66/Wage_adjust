**1. 创建 `backend/app/schemas/task.py`** (per D-05, D-06)

新建 Pydantic schema 文件：

```python
from __future__ import annotations
from typing import Any, Optional
from pydantic import BaseModel

class TaskStatusResponse(BaseModel):
    task_id: str
    status: str  # 'pending' | 'running' | 'completed' | 'failed'
    progress: Optional[dict] = None  # {processed: int, total: int, errors: int}
    result: Any = None  # evaluation dict 或 import summary dict (per D-06)
    error: Optional[str] = None

class TaskTriggerResponse(BaseModel):
    task_id: str
    status: str  # always 'pending' on trigger
```

**2. 创建 `backend/app/tasks/evaluation_tasks.py`** (per D-07, D-09)

- 导入 `celery_app`, `SessionLocal`, `get_settings`
- 装饰器：`@celery_app.task(name='tasks.generate_evaluation', bind=True, autoretry_for=(Exception,), max_retries=2, retry_backoff=True, soft_time_limit=300, time_limit=360)`
- 函数签名：`def generate_evaluation_task(self, submission_id: str, force: bool = False) -> dict`
- 函数体：
  1. `db = SessionLocal()` + `try/finally db.close()`
  2. `self.update_state(state='STARTED', meta={'status': 'running'})` (per D-07 粗粒度状态)
  3. `settings = get_settings()`
  4. `service = EvaluationService(db, settings)`
  5. `evaluation = service.generate_evaluation(submission_id, force=force)`
  6. 使用 `serialize_evaluation(evaluation)` 获取 `EvaluationRead`，然后 `.model_dump(mode='json')` 序列化
  7. 返回 `{'status': 'completed', 'result': serialized_dict}`
  8. 在 except 块中：如果 `self.request.retries >= self.max_retries`，返回 `{'status': 'failed', 'error': str(exc)}`；否则 `raise` 触发重试
- 在 task 函数中存储 operator 信息到 meta：`self.update_state(state='STARTED', meta={'status': 'running', 'user_id': user_id})`，函数签名增加 `user_id: Optional[str] = None` 参数

**3. 创建 `backend/app/tasks/import_tasks.py`** (per D-08, D-10, D-11)

- 装饰器：`@celery_app.task(name='tasks.run_import', bind=True, autoretry_for=(Exception,), max_retries=2, retry_backoff=True, soft_time_limit=600, time_limit=660)`
- 函数签名：`def run_import_task(self, import_type: str, file_bytes_b64: str, file_name: str, operator_id: Optional[str] = None, operator_role: Optional[str] = None) -> dict`
- 函数体：
  1. `db = SessionLocal()` + `try/finally db.close()`
  2. `import base64; raw_bytes = base64.b64decode(file_bytes_b64)`
  3. 创建一个 `io.BytesIO(raw_bytes)` 对象，构造一个简单的 file-like 对象用于 ImportService
  4. 由于 `ImportService.run_import()` 接收 `UploadFile`，需要创建一个适配器：使用 `UploadFile(file=io.BytesIO(raw_bytes), filename=file_name)` 构造
  5. `service = ImportService(db, operator_id=operator_id, operator_role=operator_role)`
  6. 在调用 `service.run_import()` 前后，使用 `self.update_state(state='PROGRESS', meta={'processed': 0, 'total': 0, 'errors': 0})` 初始化进度
  7. 注意：当前 `run_import()` 是一个整体方法，内部逐行处理。为了支持进度上报（per D-08），需要在 task 中拆分处理逻辑。**但为避免大规模重构 ImportService**，采用以下策略：
     - 直接调用 `service.run_import(import_type=import_type, upload=upload_file)`
     - 在调用前 update_state PROGRESS {processed: 0, total: 0}
     - 在调用后从返回的 ImportJob 中读取 total_rows/success_rows/failed_rows
     - 返回 `{'status': 'completed', 'result': ImportJobRead.model_validate(job).model_dump(mode='json')}`
  8. 对于更细粒度的进度上报，在 ImportService 中注入一个可选的 progress_callback 参数。在 ImportService._process_rows() 循环中每 50 行调用一次 callback。task 传入 `lambda processed, total, errors: self.update_state(state='PROGRESS', meta={'processed': processed, 'total': total, 'errors': errors})`。如果 ImportService 没有 `_process_rows` 方法，则在其主处理循环中添加 callback 调用点。**具体做法**：给 `ImportService.run_import()` 添加可选参数 `progress_callback: Optional[Callable[[int, int, int], None]] = None`，在处理每批 50 行后调用 `if progress_callback: progress_callback(processed_count, total_rows, error_count)`。
  9. 异常处理同 evaluation_tasks：达到 max_retries 时返回 failed dict，否则 raise

**4. 创建 `backend/app/api/v1/tasks.py`** (per D-05, D-06)

- `router = APIRouter(prefix='/tasks', tags=['tasks'])`
- `@router.get('/{task_id}', response_model=TaskStatusResponse)`
- 函数 `def get_task_status(task_id: str, current_user: User = Depends(get_current_user)) -> TaskStatusResponse`
- 使用 `AsyncResult(task_id, app=celery_app)` 获取结果
- 状态映射：
  - `PENDING` -> `{'task_id': task_id, 'status': 'pending'}`
  - `STARTED` -> `{'task_id': task_id, 'status': 'running'}`
  - `PROGRESS` -> `{'task_id': task_id, 'status': 'running', 'progress': result.info}` (info 包含 processed/total/errors)
  - `SUCCESS` -> 从 `result.result` 读取 dict，返回 `{'task_id': task_id, 'status': payload.get('status', 'completed'), 'result': payload.get('result'), 'error': payload.get('error')}`
  - `FAILURE` -> `{'task_id': task_id, 'status': 'failed', 'error': str(result.result)}`
  - 其他 -> `{'task_id': task_id, 'status': result.state.lower()}`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> /tasks/{task_id} | 任意认证用户可查询任意 task_id 的结果 |
| client -> /evaluations/generate | 认证用户触发异步评估任务 |
| client -> /imports/jobs | 认证用户触发异步导入任务 |
| API -> Celery broker | 任务参数通过 Redis 传递 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | Information Disclosure | GET /tasks/{task_id} | mitigate | 在 evaluation_tasks 和 import_tasks 中将 user_id 存入 task meta；轮询端点验证 current_user.id 与 meta 中 user_id 匹配，admin 角色可查看所有任务 |
| T-22-02 | Denial of Service | POST /evaluations/generate | accept | 当前用户量级小（企业内部系统），暂不限制并发任务数。后续可在 Phase 23+ 添加限流 |
| T-22-03 | Tampering | Celery task 参数 | accept | Redis broker 在内网运行，task 参数仅包含 submission_id/file_bytes 等非敏感数据，且 task 内部重新校验 DB 数据 |
| T-22-04 | Information Disclosure | import_tasks file_bytes_b64 | accept | 文件内容在 Redis 消息中短暂停留（task_acks_late=True），Redis 仅内网访问，result_expires=3600 自动清理 |
</threat_model>
