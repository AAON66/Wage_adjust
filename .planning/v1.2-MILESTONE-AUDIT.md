## Tech Debt Summary

### Phase 23: 调薪资格导入管理
1. **InMemoryRateLimiter 重复定义** — `llm_service.py` 本地定义了 InMemoryRateLimiter 类，未从 `core/rate_limiter.py` 导入。FeishuService 正确使用 core 版本，但两份实现存在维护风险。
2. **FeishuSyncPanel 轮询模式不一致** — 使用手写 `setTimeout` 递归轮询而非共享 `useTaskPolling` hook。导致：(a) 不显示同步过程中的进度；(b) 缺少 `cancelled` flag 清理机制；(c) 轮询间隔可能漂移。功能上同步可完成且结果统计正确显示。

### Phase 18: Python 3.9 兼容
3. **Python 3.9 运行时验证** — 所有代码级降级已完成（0 处 PEP 604/585 残留），但开发环境为 Python 3.14，需在目标 3.9 环境人工确认。

### Phase 24: 生产部署
4. **Docker 运行时验证** — 配置文件结构验证通过，但实际 Docker 构建和容器编排需人工运行。
